Git Branch Naming: Master vs Main

The terms "master" and "main" refer to the default branch names in Git repositories. Here’s the difference between them:

1. Historical Context

Master: Traditionally, master was the default branch name when you initialized a Git repository. This has been the case since Git's inception in 2005.

Main: Starting around mid-2020, many organizations and platforms (including GitHub) began shifting the default branch name from master to main. This change was part of a broader effort to adopt more inclusive language in software and technology.

2. Functionality

From a technical standpoint, there is no difference in functionality between master and main. They are just names for branches and can be used interchangeably. The key point is that one is typically the primary or default branch where the main codebase resides.

3. Which One to Use

New Repositories: If you create a new repository, especially on platforms like GitHub, the default branch will likely be main.

Existing Repositories: Older repositories might still use master as the default branch name.

Consistency: If you’re working on a project with an existing repository, it’s important to use the same branch name as the rest of your team or project.

4. Changing the Default Branch Name

If you want to change an existing repository from master to main, you can rename the branch using the following commands:

git branch -m master main

git push -u origin main

git push origin --delete master

After renaming, you may also need to update the default branch setting in your repository on the hosting platform.
